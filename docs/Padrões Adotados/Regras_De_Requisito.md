# Padrões para Especificação de Requisitos

Este documento define os critérios para a verificação e análise de requisitos do projeto.

## Nomenclatura

Todos os requisitos devem seguir a nomenclatura RF (Requisito Funcional) e RNF (Requisito Não-Funcional).

## Critérios de Qualidade (Baseado em Rogera Magela)

1.  **Clareza:** O requisito deve ser fácil de entender.
2.  **Completo:** O requisito deve conter toda a informação necessária.
3.  **Testável:** Deve ser possível criar um teste que verifique se o requisito foi atendido.
4.  **Rastreável:** O requisito deve ser identificável (ex: RF01, RF02...).
